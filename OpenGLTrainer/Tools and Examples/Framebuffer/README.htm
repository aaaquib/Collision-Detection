Exercise 1

Study the code in Main.cpp and the documentation on the Framebuffer class (See the OpenGLTrainer/README.html).   Compile and run the code.  

 

After playing, exit the program.    Now:

 

Place 2 breakpoints below the lines with the following comments:

 

            //- BKPT 1

and

//- BKPT 2

 

 

Rerun the program.    When the program stops at the breakpoint open the Immediate debug window (Menubar->Debug:Windows:Immediate).  In the Immediate window type:

            framebuffer->drawImmediately();

 

Continue [i.e. press F5].

 

Most likely, the program runs slow and you will see each pixel appear one after another.

 

Wait for the program to stop at breakpoint //- BKPT 2.    Step through (F10) the nested for loops.  Watch as each pixel appears on screen immediately after framebuffer->setPixel.    Try holding down F10.

 

Continue (F5).

 

Wait for the program to stop at breakpoint //- BKPT 1.   Disable the two breakpoints and Continue (F5).   Watch how slow the program runs!

 

Enable the breakpoint //- BKPT 1

 

Wait for the program to stop at breakpoint.   Now use the Immediate window and type framebuffer->drawFastest();

 

Continue (F5).

 

The program should return to running (relatively) fast.

 

As you write your rasterization algorithms you can place breaks at useful locations and use the Immediate Window  to call drawFastest() or drawImmediately() to switch between the two Framebuffer::Mode to help debug your program.   Of course, you can call these functions directly in your source code too rather than using the debugger’s Immediate Window.

Screen Zoom Tools

 

Even with the DrawImmediate mode, debugging rasterization algorithms can be tricky since individual pixels are so small.   If you have a favorite application that lets you zoom in on a portion of the desktop and see the zoomed in results live, then you can use such a tool as you debug your rasterization code.  

 

There is a program, ZoomWindow, that you can compile and run in ‘Third Party Libraries\OpenGLTrainer\Tools and Examples\ZoomWindow’.   Open the project in a separate instance of MSVS 2005 and compile and run it (Use [CTRL-F5] to run without debugging).   ZoomWindow isn’t particularly fast, however, on many machines.   See the ZoomWindow README.htm for more details.
